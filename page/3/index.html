P3375 【模板】KMP

题目描述

给出两个字符串 $s_1$ 和 $s_2$，若 $s_1$ 的区间 $[l, r]$ 子串与 $s_2$ 完全相同，则称 $s_2$ 在 $s_1$ 中出现了，其出现位置为 $l$。
现在请你求出 $s_2$ 在 $s_1$ 中所有出现的位置。

定义一个字符串 $s$ 的 border 为 $s$ 的一个非 $s$ 本身的子串 $t$，满足 $t$ 既是 $s$ 的前缀，又是 $s$ 的后缀。
对于 $s_2$，你还需要求出对于其每个前缀 $s’$ 的最长 border $t’$ 的长度。

输入格式

第一行为一个字符串，即为 $s_1$。
第二行为一个字符串，即为 $s_2$。

输出格式

首先输出若干行，每行一个整数，按从小到大的顺序输出 $s_2$ 在 $s_1$ 中出现的位置。
最后一行输出 $|s_2|$ 个整数，第 $i$ 个整数表示 $s_2$ 的长度为 $i$ 的前缀的最长 border 长度。

输入输出样例 #1

输入 #1

输出 #1

说明/提示

样例 1 解释

。

对于 $s_2$ 长度为 $3$ 的前缀 ABA，字符串 A 既是其后缀也是其前缀，且是最长的，因此最长 border 长度为 $1$。

思路

一、题目要求

  1. 给定文本串 s1 和模式串 s2。
  2. 输出 s2 在 s1 中所有出现的位置（下标从 1 开始）。
  3. 输出 s2 的每个前缀的最长 border 长度（border = 既是前缀又是后缀且不等于原串的子串）。

二、KMP 核心思想

1. 前缀函数（next 数组）

  • 定义 b[i] = 长度为 i+1 的前缀的 最长 border 长度。
  • 递推求解：
    • 用两个指针 i（当前前缀末尾）和 j（候选 border 长度）。
    • 如果 s2[i] == s2[j]，则 b[i] = ++j。
    • 否则 j = b[j-1]，继续尝试。

作用：匹配失败时，不用回退到模式串开头，而是跳到最长 border 的位置继续匹配。

2. 匹配过程

  • 用 j 表示当前匹配到的 s2 的长度。
  • 遍历 s1 的每个字符 s1[i]：
    • 若 s1[i] == s2[j]，则 j++。
    • 否则 j = b[j-1] 回退。
  • 如果 j == len2，说明匹配成功：
    • 输出起始位置 i - len2 + 2（因为 i 从 0 开始，位置从 1 开始）。
    • 然后 j = b[j-1] 继续寻找下一个匹配。

三、为什么 KMP 高效？

  • 暴力匹配：O(n * m)，每次匹配失败只移动一位。
  • KMP：O(n + m)
    • 利用 border 信息避免重复比较。
    • 匹配失败时，模式串向右移动的距离由 b[] 决定，而不是仅仅 1 位。

四、你的代码逻辑总结

五、注意事项

  • 位置计算：i 是 0-based 索引，题目要求输出 1-based 位置，所以起始位置 = i - len2 + 2。
  • 匹配成功后 j = b[j-1] 是为了重叠匹配（例如 s1="AAAA", s2="AA" 时能找到位置 1、2、3）。
  • 输出格式：最后一行每个数字后有一个空格（样例如此），你代码已经满足。

六、一句话总结

KMP 通过预处理模式串的 border 信息（前缀函数），在匹配失败时快速回退，实现线性时间匹配，同时正好求出每个前缀的最长 border 长度。

红黑树

红黑树的性质

  1. 二叉搜索树性质
    红黑树是一棵平衡二叉搜索树，其中序遍历单调不减（即中序遍历得到的序列是非递减的）。

  2. 节点颜色
    每个节点要么是红色，要么是黑色。

  3. 根节点颜色
    根节点是黑色的。

  4. 叶节点（外部节点）颜色
    每个叶节点（即 NULL 节点，也称为外部节点，目的是将红黑树变为真二叉树）是黑色的。

  5. 红色节点的子节点颜色
    每个红色节点的两个子节点都是黑色的。
    （换句话说，从每个叶子到根的所有路径上不能有两个连续的红色节点。）

  6. 黑色节点数量一致
    从根节点到每个叶子的所有路径都包含相同数目的黑色节点，这个数值称为黑高度。

对于节点 u，有左右儿子 l 和 r：

假设：

  • 左子树保留 i 条边（包含 l 到其子节点的边）
  • 右子树保留 j 条边（包含 r 到其子节点的边）
  • 加上 u 到 l 的边（边权 wl）
  • 加上 u 到 r 的边（边权 wr）

总保留边数：

total = i + j + (选左儿子 ? 1 : 0) + (选右儿子 ? 1 : 0)

总苹果数：

苹果 = (选左儿子 ? dp[l][i] + wl : 0) + (选右儿子 ? dp[r][j] + wr : 0)

遍历所有 i, j 组合，取最大值。

4. 实现细节

技巧：用 i = -1 表示不选该子节点

其中 i + j + 2 的含义：

  • 如果选左儿子：i 包含左子树内的边，加上当前边 → i + 1
  • 如果选右儿子：j 包含右子树内的边，加上当前边 → j + 1
  • 两者相加：(i+1) + (j+1) = i + j + 2
  • 如果不选某儿子，对应 i = -1 或 j = -1，代入后自动消去

5. 复杂度

  • 状态数：O(N × Q)
  • 转移：O(Q²)
  • 总复杂度：O(N × Q²)，N, Q ≤ 100，可行

6. 关键点总结

要点 说明
二叉树 每个节点最多 2 个子节点
保留边必须连通根 不能孤立选边
分组背包 左右子树是两组，枚举分配保留的边数
i = -1 技巧 优雅处理不选某子树的情况
边数计算 siz[u] = siz[l] + siz[r] + 2（若两子都存在）

题目描述

小 A 的工作不仅繁琐，更有苛刻的规定，要求小 A 每天早上在 $6:00$ 之前到达公司，否则这个月工资清零。可是小 A 偏偏又有赖床的坏毛病。于是为了保住自己的工资，小 A 买了一个空间跑路器，每秒钟可以跑 $2^k$ 千米（$k$ 是任意自然数）。当然，这个机器是用 longint 存的，所以总跑路长度不能超过 maxlongint 千米。小 A 的家到公司的路可以看做一个有向图，小 A 家为点 $1$，公司为点 $n$，每条边长度均为一千米。小 A 想每天能醒地尽量晚，所以让你帮他算算，他最少需要几秒才能到公司。数据保证 $1$ 到 $n$ 至少有一条路径。

输入格式

第一行两个整数 $n,m$，表示点的个数和边的个数。

接下来 $m$ 行每行两个数字 $u,v$，表示一条 $u$ 到 $v$ 的边。

输出格式

一行一个数字，表示到公司的最少秒数。

输入输出样例 #1

输入 #1

输出 #1

思路

1 秒可以走 2^k 步，所以如果两个点之间恰好存在一条长度为 2 的幂 的路径，就能 1 秒 到达。

算法步骤

步骤 做什么
1 用倍增 DP 找出所有长度为 2^i 的路径（f[i][u][v]=1 表示 u 到 v 存在长 2^i 的路径）
2 把这些点对之间的距离设为 1（因为 1 秒就能到）
3 用 Floyd 求 1 号点到 n 号点的最短路

举例

  • 原始边：1→1, 1→2, 2→3, 3→4
  • 路径 1→2→3→4 长度为 3，不是 2 的幂，不能 1 秒
  • 加上自环 1→1，可以走 1→1→2→3→4，长度 = 4 = 2²，所以 1 秒 到达

一句话总结

倍增找 2 的幂路径 → 缩成 1 秒边 → Floyd 跑最短路
