D

题意

给定字符串 S 和 T，问 S 的所有非空子串中，有多少个子串不以 T 作为子序列（不要求连续）。

关键点：

  • 子串是连续的，子序列可以不连续。
  • 两个子串只要位置不同就算不同（即使内容相同）。
  • 长度限制：|S| ≤ 2×10^5，|T| ≤ 50。

换句话说，要统计所有子串 [l, r]（1 ≤ l ≤ r ≤ |S|），使得 不存在一种方式从 S[l..r] 中按顺序挑出字符等于 T。

更直白地：子串内不能按顺序匹配出整个 T。

思路

答案 = 总子串数 − 包含 T 的子串数

关键步骤

  1. 预处理 nxt[i][c]：从位置 i 开始，字符 c 首次出现的位置（O(26n)）

  2. 对每个左端点 i：

    • 贪心匹配 T：pos = nxt[pos][t[j]]; pos++
    • 若匹配成功，cnt += n - pos + 1（以 i 为左端点的合法右端点个数）

  3. 输出：n*(n+1)/2 - cnt

题意总结

给定两个正整数序列 $A=(A_1,\dots,A_N)$ 和 $B=(B_1,\dots,B_M)$，求：

$$\sum_{i=1}^{N} \sum_{j=1}^{M} A_i \cdot B_j \cdot (i \bmod j) \pmod{998244353}$$

约束：$N,M \le 5\times 10^5$，$A_i,B_j \le 5\times 10^5$

解题思路

核心观察

  1. 模运算展开：$i \bmod j = i - j \cdot \lfloor i/j \rfloor$

  2. 按商分组：对于固定的 $j$，当 $\lfloor i/j \rfloor = k$ 时，$i$ 的范围是 $[jk, \min(j(k+1)-1, N)]$，在此范围内 $i \bmod j = i - jk$

  3. 区间求和转化：
    $$\sum_{i=l}^{r-1} A_i \cdot (i - jk) = \sum_{i=l}^{r-1} A_i \cdot i - jk \cdot \sum_{i=l}^{r-1} A_i$$

  4. 前缀和优化：预处理两个前缀和数组，可在 $O(1)$ 时间内计算区间和：

    • sum[x] = $\sum_{k=0}^{x-1} A_k$
    • sum2[x] = $\sum_{k=0}^{x-1} A_k \cdot k$

  5. 复杂度分析：对于每个 $j$，$k$ 的范围是 $0 \sim \lfloor N/j \rfloor$，总枚举次数为 $\sum_{j=1}^M \lfloor N/j \rfloor = O(N \log N)$（调和级数）

AtCoder ABC 452 F - Interval Inversion Count

📋 题意简述

给定长度为 $N$ ($N \leq 5 \times 10^5$) 的排列 $P$ 和整数 $K$，求有多少个区间 $[l, r]$ 满足：该区间内子序列的逆序对数量恰好等于 $K$。

💡 解题思路

核心转化

直接统计「恰好 $K$ 个逆序对」较难，转化为：
$$\text{ans} = f(K) - f(K-1)$$
其中 $f(k)$ 表示逆序对数 $\leq k$ 的区间数量。

单调性 & 双指针

关键观察：区间缩短，逆序对数不会增加。

  • 若 $[l, r]$ 逆序对 $\leq K$，则 $[l+1, r]$ 和 $[l, r-1]$ 也 $\leq K$
  • 因此对每个左端点 $l$，满足条件的右端点 $r$ 具有单调性 → 可用滑动窗口（双指针）

逆序对动态维护（Fenwick Tree）

窗口 $[l, r)$ 内的逆序对动态更新：

操作 逆序对变化 BIT 查询
右扩 $r$ $+$ 窗口中 $> P[r]$ 的元素个数 bit.sum(P[r], N)
左缩 $l$ $-$ 窗口中 $< P[l]$ 的元素个数 bit.sum(0, P[l])

💻 代码详解（带注释）

🔑 关键细节

  1. BIT 查询范围：

    • bit.sum(a, b) 查询值域 $[a, b)$ 的元素个数
    • 加 $P[r]$ 时：查 $[P[r], N)$ 即大于 $P[r]$ 的值 → 新增逆序对
    • 减 $P[l]$ 时：查 $[0, P[l])$ 即小于 $P[l]$ 的值 → 减少逆序对

  2. 时间复杂度：$O(N \log N)$

    • 双指针每个元素最多进出窗口一次：$O(N)$
    • 每次 BIT 操作 $O(\log N)$

  3. 边界处理：$K=0$ 时 $f(-1)=0$，需特判避免下溢

🧪 样例验证（Sample 1）

题意总结

定义：一个正整数序列 $X$ 被称为 221 序列，当且仅当对其游程编码（Run-Length Encoding, RLE）后的每一段 $(值, 长度)$ 都满足 值 = 长度。

  • 例如 (2,2,3,3,3,1) 是 221 序列，因为 RLE 为 (2,2), (3,3), (1,1)，每段值都等于长度。
  • (1,1) 不是，RLE 为 (1,2)，$1 \ne 2$。

问题：给定长度为 $N$ 的序列 $A$（$A_i \in [1,9]$），求 $A$ 中有多少个 不同的非空连续子序列 是 221 序列。

  • 即使子序列出现在不同位置，只要序列内容相同只计一次。

约束：$N \le 5 \times 10^5$，$A_i \le 9$。

G - 221 Substring

解题思路

1. 游程编码分析

对 $A$ 进行游程编码，得到块序列 $(v_1, m_1), (v_2, m_2), \dots, (v_k, m_k)$，其中 $v_i$ 是值，$m_i$ 是连续出现次数。

一个子序列是 221 序列，当且仅当它对应 RLE 的一段区间 $[l, r]$，且满足：

  • 内部块（$l < i < r$）：必须完整保留，且 $v_i = m_i$。
  • 边界块（$i = l$ 或 $i = r$）：可以截取一部分。若截取长度为 $L$，需满足 $L = v_i$，这要求 $v_i \le m_i$。
    • 左边界可取后缀长度为 $v_l$ 的部分。
    • 右边界可取前缀长度为 $v_r$ 的部分。

因此，一个合法的 221 子序列由值序列 $(v_l, v_{l+1}, \dots, v_r)$ 唯一确定，且需满足：

  • 对所有 $i \in (l, r)$，$v_i = m_i$。
  • $v_l \le m_l$ 且 $v_r \le m_r$。
  • 若 $v_i > m_i$，该块无法参与任何 221 序列。

2. 问题转化

构造一个新序列 $T$，将每个 RLE 块映射如下：

  • 若 $v_i > m_i$：映射为 (0)（表示断裂，无法使用）。
  • 若 $v_i = m_i$：映射为 (v_i)（可作为内部或边界，可连接）。
  • 若 $v_i < m_i$：映射为 (v_i, 0, v_i)。
    • 第一个 $v_i$ 可作为右边界（与其左侧连接）。
    • 第二个 $v_i$ 可作为左边界（与其右侧连接）。
    • 中间的 0 阻止左右穿透，保证该块不能同时作为左右边界或作为内部块。

关键结论：$A$ 中不同的 221 子序列与 $T$ 中 不包含 0 的不同子数组 一一对应。

  • 因为 221 序列的结构由值序列唯一确定，计数不同 221 序列等价于计数 $T$ 中不含 0 的不同子数组。

3. 算法实现

问题转化为：求序列 $T$ 中不含 0 的不同子数组个数。

使用 后缀数组 (Suffix Array, SA) 和 LCP 数组：

  • 设 zerofree[i] 表示从位置 $i$ 开始向后连续不含 0 的最大长度。
  • 对后缀数组中第 $k$ 个后缀（起始位置为 SA[k]），其能贡献的不含 0 的新子串数量为：
    $$\max(0, \text{zerofree}[\text{SA}[k]] - \text{LCP}[k])$$
    其中 LCP[k] 是该后缀与前一个后缀的最长公共前缀长度，需减去重复部分。
  • 答案为所有后缀贡献之和。

复杂度：

  • 构造 $T$：$O(N)$。
  • 后缀数组与 LCP：$O(|T|)$（利用字母表大小 $\sigma=10$ 可线性构造）。
  • 总时间复杂度 $O(N)$，空间 $O(N)$。

代码注释解析

代码关键点说明

  1. maximals 构造逻辑：
    • 当遇到 val < len 的块时，区间包含它并停止扩展，但 idxlen 更新为 jlen（未越过该块）。

蓝桥杯2026

B

题意

这道题要求计算在 $2048 \times 2048$ 的网格中，有多少种将每个格子填为 $L$ 或 $Q$ 的方案，满足三个条件：

  1. 每行 $Q$ 的个数为奇数
  2. 每列 $Q$ 的个数为奇数
  3. 任意 $2\times2$ 子区域中 $Q$ 的个数为奇数

你需要输出结果对 $998244353$ 取模。

思路

打表

n 为奇数时只有 1 种方案，n 是 4 的倍数时，注意到 32 和 8192 都是 2 的幂，可以归纳出来这种时候是 2 ^ (2n−3)，其他情况下的 n 的答案是 0。题目要求的 n=2048，是 4 的倍数，所以答案就是 2 ^ 4093 取模。

推理

由于每个 2×2 的矩阵的和都是奇数，即 a 1,1​+a 1,2​+a 2,1​+a 2,2​是奇数且 a 2,1​+a 2,2​+a 3,1​+a 3,2​也是奇数，于是我们知道 a 1,1​+a 1,2​和 a 3,1​+a 3,2​同奇偶。

推广出来，我们发现 a i,j​+a i+1,j​和 a i,j−2​+a i+1,j−2​同奇偶，且 a i,j​+a i,j+1​与 a i+2,j​+a i+2,j+1​同奇偶。

如果我们已知第一行和第一列的所有数，根据这个规律可以把整个矩阵填满（首先 a 1,1​,a 1,2​,a 2,1​已知，奇偶性可以算出 a 2,2​，然后利用隔行隔列同奇偶的性质可以填满整个矩阵，可以手玩一下）。

同时如果保证第一行的和是奇数的话，则由于第三行每相邻两个数的和的奇偶性和第一行相同，因此所有奇数行的和肯定也是奇数；由于每个 2×2 的矩阵的和都是奇数，因此前两行的和一定是偶数（前两行可以分割成 1024 个 2×2 的矩阵），因此第二行的和也是奇数，因此偶数行的和也是奇数。

同理只要第一列的和是奇数，所有列的和都是奇数。

怎么保证第一行第一列的和是奇数？只填 2047 个，算算奇偶性决定最后一个怎么填。

于是一共有 2047+2047−1=4093 个位置可以随便填，答案就是 2 4093 。减一因为 a 1,1​被算了两次。

题意

午高峰有 $N$ 个订单，必须按固定顺序依次配送，可分成若干批次。站点有 $M$ 种交通工具，每种具属性 $A_i$（路途耗时系数）和 $B_i$（拥挤系数）。

对于一批含 $L$ 个订单，选用第 $i$ 种工具，耗时为：

  1. 路途：$L \times A_i$
  2. 取餐：$\frac{L(L-1)}{2} \times B_i$（处理挤压关系）
  3. 折返：首批为 $0$，之后每新开一批需 $+X$

求送完 $N$ 单的最小总耗时。

输入：$N, M, X$ 及 $M$ 行 $(A_i, B_i)$
输出：最小总耗时（整数）
范围：$N, M \le 5000$，$A_i, B_i, X \le 10^9$

思路

1. 预处理最优单车耗时

对每个可能的批次长度 $l \in [1, N]$，枚举 $M$ 种车，计算该长度下的最小基础耗时（不含折返）：
$$c[l] = \min_{1 \le i \le M} \left( l \cdot A_i + \frac{l(l-1)}{2} \cdot B_i \right)$$

2. 动态规划

设 $dp[i]$ 表示送完前 $i$ 单的最短时间。

枚举最后一批的起点 $j$（即最后一批为 $[j+1, i]$，长度 $i-j$）：

  • 若 $j=0$（整批为第一批）：耗时 $c[i]$
  • 若 $j>0$：耗时 $dp[j] + c[i-j] + X$

转移方程：
$$dp[i] = \min\left(c[i],\ \min_{1 \le j < i} (dp[j] + c[i-j] + X)\right)$$

技巧：初始化 dp[i] = c[i] 可自然处理 $j=0$ 的边界，无需特判。

3. 复杂度

  • 预处理：$O(NM)$
  • DP：$O(N^2)$
  • 总计：$O(N^2 + NM)$，$N,M \le 5000$ 可过

题意

给定长度为 $N$ 的序列 $A$，选择三个分割点 $1 \le i < j < k < N$，将序列切成 4 段非空连续子段。

目标：最大化
$$|\text{sum}_1| + |\text{sum}_2| + |\text{sum}_3| + |\text{sum}_4|$$

其中 $\text{sum}_t$ 表示第 $t$ 段的元素和。

输入：$N$ 及序列 $A_1 \dots A_N$
输出：最大绝对值和
范围：$4 \le N \le 10^5$，$|A_i| \le 10^9$

思路

方法一：枚举符号 + 线性 DP（O(16N)，推荐⭐）

核心洞察

绝对值 $|x| = \max(x, -x)$。若我们提前假设每段和的符号（正/负），绝对值即可直接去掉。

枚举 4 段符号的所有 $2^4=16$ 种组合，对每种组合：

  • 若第 $t$ 段假设符号为 $sgn[t] \in {+1,-1}$
  • 目标变为：$sgn[1]\cdot S_1 + sgn[2]\cdot S_2 + sgn[3]\cdot S_3 + sgn[4]\cdot S_4$

为什么正确？

若某段实际和与假设符号相反，该项贡献会比真实 $|x|$ 小 $2|x|$，结果一定 $\le$ 真实答案。枚举所有组合取 $\max$，必能命中最优符号分配。

DP 转移

dp[i][k]：前 $i$ 个数分成 $k$ 段，在当前符号组合下的最大带符号和。

$$dp[i][k] = \max(dp[i-1][k],\ dp[i-1][k-1]) + sgn[k] \cdot A[i]$$

  • dp[i-1][k]：$A[i]$ 接在第 $k$ 段末尾
  • dp[i-1][k-1]：$A[i]$ 作为第 $k$ 段开头

复杂度

$16 \times 4 \times N = O(64N)$，常数极小，代码简洁。

方法二：拆绝对值 + 前缀最大值（O(4N)，理论最优）

核心转化

设前缀和 $S[i] = \sum_{j=1}^i A[j]$，第 $[j+1, i]$ 段和为 $S[i]-S[j]$。

转移方程：
$$f[i][k] = \max_{j < i} \left( f[j][k-1] + |S[i] - S[j]| \right)$$

拆绝对值：

  • 若 $S[i] \ge S[j]$：$f[j][k-1] - S[j] + S[i]$
  • 若 $S[i] < S[j]$：$f[j][k-1] + S[j] - S[i]$

只需维护两个全局最大值：

  • max1 = max(f[j][k-1] - S[j])
  • max2 = max(f[j][k-1] + S[j])

复杂度

$4 \times N = O(N)$，理论最快，但边界处理需谨慎。

方法三：线段树优化（O(N log N)，通用扩展）

适用场景

当题目增加区间修改、动态查询等复杂条件时，前缀最大值法失效，线段树可灵活应对。

核心思想

仍基于拆绝对值，用线段树按 $S[j]$ 的值域维护：

  • m1: $\max(f[j][k-1] - S[j])$（用于 $S[i] \ge S[j]$）
  • m2: $\max(f[j][k-1] + S[j])$（用于 $S[i] < S[j]$）

配合离散化 + 懒标记实现高效查询更新。

B. OIE Excursion

题意

1. 问题重述

  • 有 ( n ) 个位置，第 ( i ) 个位置有一个志愿者，计时器初始值 ( a_i )。
  • 计时器每秒钟加 1，到 ( m ) 就归 0（模 ( m ) 加法）。
  • 在第 ( x ) 秒，第 ( i ) 位志愿者的计时器值是 ((a_i + x) \bmod m)。
  • 志愿者只在计时器 = 0 的时候警惕（会抓赫克托），其他时候不看。
  • 赫克托从位置 0 出发，想到位置 ( n+1 )。
  • 每秒 结束 时他可以：左移一步、右移一步、或不动。
  • 被抓的条件：在某秒开始时，他在位置 ( i ) ((1 \le i \le n))，且第 ( i ) 个志愿者在那秒开始时计时器 = 0。

注意关键细节：

  • 每 秒开始 时，先检查抓不抓，然后这一秒赫克托移动，然后到下一秒开始，循环。
  • 时间 ( x ) 从 0 开始，第 0 秒开始时第 ( i ) 的计时器 = ( a_i )。
  • 当某个志愿者 ( i ) 的计时器等于 0 的时候，赫克托不能在这一秒的开始位于 ( i )。

思路

如果存在一个长度≥ m 的连续等值子数组，则不可能逃脱。这是因为穿越这样的区块至少需要 m 秒，而每个位置在任何连续的 m 秒内至少会向 0 方向移动一次。
否则，逃逸是可能的。我们可以把数组看成连续的等值区块，每个区块的长度都小于 m。对于每个区块来说，都存在一个小于 m 的时间窗口，在这个窗口内，没有任何一个位置会看向 0。由于两个连续的区块具有不同的值，因此它们的危险时刻永远不会重合。

题意概括

用 p 个单位线段 + q 个 L 形块（每个占 2 条边），恰好拼成 n×m 网格的所有边，求任意合法 n, m 或输出 -1。

D Unique Values

题意概括

有一个长度为 $2n+1$ 的隐藏数组，包含 $1$ 到 $n$ 的整数。其中有一个数出现了 3次，其余所有数均出现 2次。
你需要通过最多 66次 交互查询找出那个出现3次的数的三个位置索引。

查询方式：
给定一个下标集合 $S$，返回该集合中恰好出现一次的不同数值的个数。

解题思路

核心在于利用查询结果推断目标元素（出现3次的数）在集合 $S$ 中的数量（0, 1, 2, 或 3）。

设目标数为 $x$，其在集合 $S$ 中出现次数为 $k$ ($0 \le k \le 3$)。
令 $T$ 为 $S$ 的补集。
查询 $S$ 得到 $ans_S$，查询 $T$ 得到 $ans_T$。

关键性质推导：

  1. 总数关系：整个数组中，除了 $x$ 以外其他数都成对出现。
    • 若 $k=0$ 或 $k=3$：$x$ 在 $S$ 中要么没出现，要么全出现（3次）。此时 $S$ 中 $x$ 的贡献为0（因为出现次数>1）。$S$ 和 $T$ 中其他数的分布是对称的（成对拆分），但由于总长度奇偶性，需结合具体计算。
    • 更简单的判断逻辑（参考 tourist 代码逻辑）：
      • 如果 $ans_S == ans_T$：说明 $S$ 中包含的目标数个数是 0 或 3。
        • 如何区分？看 $|S| - ans_S$ 的奇偶性。
        • 集合 $S$ 的总元素数 = (出现1次的数) + (出现2次的数 $\times$ 2) + (出现3次的数 $\times$ $k$)。
        • $|S| - ans_S = 2 \times (\text{成对数}) + 3k$ (若$k=3$则贡献9，若$k=0$贡献0)。
        • 实际上，若 $k=0$，剩余都是成对的，差值为偶数。若 $k=3$，贡献 $3 \times 3 = 9$ (奇数)？不对，这里逻辑是：
        • 正确逻辑：$S$ 中非唯一值元素的总数 = $|S| - ans_S$。这些元素由成对的数和 $x$ 组成。
        • 如果 $k=0$：全是成对的，总数是偶数。
        • 如果 $k=3$：$x$ 贡献3个，其余成对。总数 = 偶数 + 3 = 奇数。
        • 结论：若 $ans_S == ans_T$，且 $(|S| - ans_S)$ 是奇数，则 $k=3$；否则 $k=0$。
      • 如果 $ans_S \neq ans_T$：说明 $S$ 中包含的目标数个数是 1 或 2。
        • 若 $k=1$：$S$ 中有1个 $x$（不计入 $ans_S$），$T$ 中有2个 $x$（也不计入 $ans_T$）。但 $S$ 少了一个配对，导致 $ans_S$ 相对变化。
        • 直观结论：若 $ans_S > ans_T$，则 $k=1$；若 $ans_S < ans_T$，则 $k=2$。（或者反之，取决于具体实现，代码中是 x > y 返回 1）。

算法流程：

  1. 定义函数 Query(p)：查询前缀 $[1, p]$ 中目标数的个数（0~3）。
    • 构造集合 $S = [1, p]$，$T = [p+1, 2n+1]$。
    • 分别查询 $S$ 和 $T$。
    • 根据上述逻辑判断 $S$ 中有几个目标数。
  2. 三次二分查找：
    • 第一次二分：在整个范围 $[1, 2n+1]$ 中找到第一个目标数位置 $pos_1$。条件是 Query(mid) >= 1。
    • 第二次二分：忽略 $pos_1$，在剩余区间找第二个。逻辑同上，目标是找到累计第2个的位置。
    • 第三次二分：忽略前两个，找第三个。
    • 注：由于是非自适应交互器，我们可以直接对“前缀中目标数数量”进行二分。
      • 找第1个：寻找最小的 $p$ 使得 Query(p) >= 1。
      • 找第2个：寻找最小的 $p$ 使得 Query(p) >= 2。
      • 找第3个：寻找最小的 $p$ 使得 Query(p) >= 3。
  3. 每次二分需要 $\log_2(2n+1)$ 次查询，每次 Query 内部需要2次交互（查 $S$ 和 $T$）。
    • 总查询次数 $\approx 3 \times 2 \times \log_2(2001) \approx 6 \times 11 = 66$。刚好满足限制。

关键点总结：

  1. 奇偶性判断：利用 $|S| - \text{unique}$ 的奇偶性区分 0 和 3。
  2. 大小比较：利用 $ans_S$ 和 $ans_T$ 的大小关系区分 1 和 2。
  3. 二分策略：将问题转化为“前缀和”形式的单调性问题（前缀中目标数数量随长度单调不减），从而进行三次独立的二分查找。

D2 Unique Values

题意概括

有一个长度为 $2n+1$ 的秘密数组 $a$，元素为 $1$ 到 $n$ 的整数：

  • $n-1$ 个值各出现 2 次
  • 1 个值出现 3 次

目标：用最多 33 次查询找出出现三次的值的三个位置。

查询操作：选择若干索引，返回这些位置中恰好出现一次的不同值的个数。

解题思路

核心转化

定义辅助数组 $b$：

情况 $b_i$ 值
$a_i$ 是某值的第一次出现 $1$
$a_i$ 是某值的第二次出现 $-1$
$a_i$ 是某值的第三次出现 $0$

关键性质：查询前缀 ${1,2,\ldots,k}$ 的返回值 = $b_1 + b_2 + \cdots + b_k$（即 $b$ 的前缀和）。

奇偶性分析

  • 普通值（出现2次）：贡献 $1 + (-1) = 0$，不影响奇偶性
  • 特殊值（出现3次）：
    • 第1次出现：$+1$ → 奇偶性翻转
    • 第2次出现：$-1$ → 奇偶性再次翻转（恢复）
    • 第3次出现：$0$ → 奇偶性不变

因此，前缀和的奇偶性变化点就是特殊值的三个位置！

二分搜索策略

进行 3 轮二分搜索，每轮找一个位置：

  1. 查询前缀和，比较其奇偶性与期望奇偶性
  2. 若奇偶性不符，说明已越过某个特殊位置 → 向左收缩
  3. 若奇偶性符合，说明尚未到达 → 向右收缩
  4. 找到位置后，将其加入已知集合，在左侧区间继续搜索下一个

查询次数：$3 \times \lceil \log_2(2n+1) \rceil \leq 3 \times 11 = 33$，满足限制。

E. Coloring a Red Black Tree

题意

给定一棵 $n$ 个节点的树，节点初始为红(1)或黑(0)，保证至少一个红节点。
操作：选中节点 $a$，等概率随机选其邻居 $b$，将 $a$ 染成 $b$ 的颜色。
目标：全树变红。求在最优操作顺序下的最小期望操作次数。

核心思路

  1. 连通块分解：红节点视为“已感染源”，将原树切割为若干纯黑连通块。各块独立求解，答案相加。
  2. 树形 DP：对每个黑块任选根。定义：
    • dp[u][0]：$u$ 在父节点之前变红时，清完 $u$ 子树的最小期望。
    • dp[u][1]：$u$ 在父节点之后变红时，清完 $u$ 子树的最小期望。
  3. 转移逻辑：
    • $u$ 自身变红的期望步数服从几何分布：$E = \frac{\deg(u)}{\text{当前红邻居数}}$。
    • 红邻居数 = r[u]（初始红邻居）+ i（父节点状态 0/1）+ k（已提前染红的子节点数）。
    • 贪心排序：子节点若在 $u$ 前染贡献 dp[v][0]，在后染贡献 dp[v][1]。按差值 dp[v][0]-dp[v][1] 升序排序，枚举前缀长度 $k$，取最小总代价。
  4. 统计答案：遍历所有黑块根节点，累加 dp[root][0]（块外视为已红）。

F - Avoid Division

题意

题意理解

有一棵 N 个节点的树，有 K 种颜色，颜色 i 最多能涂 C[i] 个节点。
总节点数 N ≤ Σ C[i]。

要求：
对每条边 (u, v)，把树切成两个连通分量（子树），这两个子树必须同时包含至少一个相同颜色的节点。

换句话说，对于每条边，存在至少一种颜色，这种颜色在两个子树中都有出现。

条件转化

考虑一条边 (u, v)，假设删掉这条边后，两边的节点集合分别是 S 和 T。

条件：存在颜色 c，使得 S 和 T 中都至少有 1 个节点颜色是 c。

等价于：不存在一种颜色，它完全落在其中一侧。

换句话说：对于每条边，不能出现“某种颜色的所有节点都在树的同一侧”的情况。

关键观察

假设我们给每个节点染了一种颜色。
那么对于颜色 c，它的所有节点构成一些连通块（由于树结构，颜色 c 的节点可能分布在多个子树里，但整体在树中可能形成若干连通块）。
如果颜色 c 的所有节点都在以某个节点为根的某个子树里（切某条边使得另一侧完全没有这种颜色），那么这条边就不满足条件（对颜色 c 来说）。

因此，对每种颜色，它的节点必须“散布”在整棵树中，不能完全被某一条边割到一侧。

判定思路

对每条边 (u, v)，设子树大小为 size[u] 或 size[v]（以 1 为根时）。
删掉这条边后，两侧节点数分别为 s 和 N-s。

如果某种颜色 c 的节点数 cnt[c] ≤ s 且 cnt[c] ≤ N-s，那么它可能被完全放在一侧吗？
不，要完全放在一侧，必须满足 cnt[c] ≤ s 或 cnt[c] ≤ N-s，并且这些节点确实都在那侧。但如果我们构造时避免这种分布，就需要检查：是否存在一条边，使得 cnt[c] 全在较小的一侧？

实际上，条件可以这样理解：

对于每条边，对于每种颜色，不能出现“颜色 c 的所有节点都在边的某一侧”。

因此，对任意颜色 c，它的节点必须分布在树中，使得任何边的两侧都至少有一个该颜色的节点。

这等价于：对每种颜色 c，它的节点集是树的一个顶点覆盖？不完全是，是“节点集不能完全落在某个子树内”。

更精确的判定

对边 e，如果所有颜色中，存在至少一种颜色 c 满足：两侧都有颜色 c，则 e 合法。

等价地，如果某条边 e 的每一侧的节点颜色集合，都不包含某一种颜色，那 e 就非法。
也就是说，如果某颜色 c 的所有节点都在其中一侧，那这条边对颜色 c 来说不满足，但对别的颜色可能满足。

所以判定：
对于每条边，不能存在一种颜色 c，使得 c 的所有节点都在该边的同一侧。

如何构造

构造方法（贪心 + DFS）：

  1. 统计每种颜色可用次数 C[i]。

  2. 从根开始 DFS，处理子树。

  3. 对每个子树，尽量让它的节点颜色与父节点侧的颜色“分散”开。

  4. 关键：如果某个颜色在当前子树的节点数 = C[i]（即它已经用完），那么它以后不能再出现，那么在树中，它的所有节点必然在同一个连通块里，这会违反哪些边？
    答案是：如果某颜色 c 的节点已经全部出现，并且这些节点都在某子树里，那么切掉该子树与父节点的边，另一侧就没有颜色 c，因此这条边必须被其他颜色覆盖。

    所以只要保证每条边至少被一种颜色覆盖即可，而每种颜色覆盖的边集是它的节点集在树中形成的路径并集。

最终简化（已知结论）

这类题的结论是：
只要对每个节点，我们分配一个颜色，使得每种颜色 c 的节点集在树中构成一个连通块，那么任意切边，如果切到的连通块内部有颜色 c，另一侧也有，那么颜色 c 就覆盖了这条边。

构造方法：

  • 对每种颜色 c，选择一个节点作为“根”节点，然后把 C[c] 个节点（包括根）染成颜色 c，要求这些节点在树中连通。
  • 这样，对每条边，如果它连接了颜色 c 的连通块内外，那么两侧都有颜色 c。

因此，我们只需要为每种颜色构造一个连通块，且所有颜色的连通块覆盖所有节点。

实现思路

  1. 把节点按 DFS 序排列。
  2. 把 C[i] 种颜色的节点在 DFS 序上连续分配（类似区间分配），这样每种颜色的节点在树中形成连通块（因为 DFS 序连续的在树中不一定连通，但这里是选 DFS 序的一段连续节点，它在树中通常是连通的吗？不是，但可以保证通过树的性质构造连通块）。
  3. 更可靠的方法：以 1 为根，DFS，每个节点分配颜色时，尽量让子节点与父节点颜色相同，直到该颜色用完。

这样可以保证每种颜色形成一个连通块。

算法步骤

  1. 读入 N, K，读入树。

  2. 读入 C[1..K]。

  3. 把颜色按 C 降序排序（大颜色先分配，因为更容易形成大块）。

  4. DFS 分配颜色：

    • 对每个节点，尝试继承父节点颜色（如果父节点颜色还有剩余次数）。
    • 否则换下一个可用颜色。

  5. 分配完后，检查每条边是否满足条件：
    对于边 (u,v)，设颜色 c 在 u 侧和 v 侧都有出现？
    更简单：对每条边，检查所有颜色是否至少有一个在两侧都有。
    复杂度 O(NK) 不行，需要优化。
    实际上，我们只需检查是否存在一种颜色 c，使得所有 c 节点都在同一侧。
    因为我们的构造保证了每种颜色的节点是连通的，所以只要检查该颜色的连通块是否完全在某个子树内（即该子树包含全部该颜色节点且子树外没有）。
    用 DFS 序 + 子树大小判断：如果某个颜色 c 的节点最小 DFS 序和最大 DFS 序在某个子树区间内且子树大小 = cnt[c]，则它全在这子树内，不满足条件。

  6. 如果检查通过，输出染色方案；否则输出 -1。

小红的部分不同字符串

描述

$\hspace{15pt}$有一个未知的长度为 $n$ 的字符串，保证该字符串仅由 $26$ 个小写英文字母组成。现有 $n$ 个限制条件，用一个数组 $a = {a_1, a_2,\dots,a_n}$ 表示，意为 $s_i \ne s_{a_i}$（即字符串中第 $i$ 个字符与第 $a_i$ 个字符不同）。

$\hspace{15pt}$请你计算在满足所有限制条件的情况下，有多少种不同的字符串。由于答案可能很大，请输出其对 $998244353$ 取模的结果。

思路

这段代码的核心思路是图论建模 + 基环树染色计数。具体步骤概括如下：

  1. 问题转化：
    将字符串的每个位置 $i$ 看作图的节点，约束 $s_i \neq s_{a_i}$ 看作节点 $i$ 与 $a_i$ 之间的无向边。由于每个节点恰好有一个 $a_i$，整个图由若干个基环树连通分量组成（即每个连通块中恰好有一个环）。

  2. 独立计算连通分量：
    不同连通分量之间的字符选择互不影响，总方案数为各分量方案数的乘积。

    • 使用 BFS 遍历未访问节点，找出当前连通分量的所有节点，得到分量大小 $S$。

  3. 确定环长 $L$：
    在找到的连通分量中，利用有向关系 next[i] = a[i] 寻找唯一的环。

    • 从分量中任意一点出发，沿 a[i] 路径行走，直到遇到重复节点，该节点即为环上节点。
    • 再次遍历该环，统计环的长度 $L$。

  4. 应用染色公式：
    对于大小为 $S$、环长为 $L$ 的基环树，用 $K=26$ 种颜色进行合法着色（相邻不同色）的方案数公式为：
    $$ W = (K-1)^S + (-1)^L \cdot (K-1)^{S-L+1} $$

    • 利用快速幂计算模意义下的幂次。
    • 根据 $L$ 的奇偶性决定第二项的符号。

  5. 累乘结果：
    将所有连通分量的方案数 $W$ 相乘，得到最终答案。

总结：通过 BFS 拆分连通块 $\rightarrow$ 追踪有向边找环长 $\rightarrow$ 代入基环树染色公式 $\rightarrow$ 累乘得解。
