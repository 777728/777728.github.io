C. The 67th Permutation Problem

题意

给你一个整数 $n$。你必须构造一个长度为 $3n$ 的排列 $^{\text{∗}}$，如果你把这个排列分成 $n$ 个连续的块，每个块有 $3$ 个元素，那么这些块的中位数之和 $^{\text{†}}$ 是最大的。

更正式地说，你必须构造一个长度为 $3n$ 的排列 $p$，使得 $\sum_{i=0}^{n-1}\operatorname{median}(a_{3i+1},a_{3i+2},a_{3i+3})$ 最大化。如果有多个可能的 $p$，请输出任意一个。

$^{\text{∗}}$ 长度为 $n$ 的排列是由 $n$ 个不同的整数组成的数组，这些整数从 $1$ 到 $n$ 按任意顺序排列。例如，$[2,3,1,5,4]$ 是一个排列，但 $[1,2,2]$ 不是一个排列（$2$ 在数组中出现了两次），$[1,3,4]$ 也不是一个排列（$n=3$ 但数组中有 $4$）。

$^{\text{†}}$ 包含 $3$ 个元素的数组 $b$ 的中位数是 $b$ 按非递减顺序排序后的第 $2$ 个元素。

思路

贪心。

E. The 67th XOR Problem

题意

给你一个数组 $a$，最初包含 $n$ 个非负整数。

你需要执行以下操作 $n-1$ 次：

选择 $a$ 的索引 $i$（$1 \leq i \leq |a|$，其中 $|a|$ 表示 $a$ 的当前长度）。让 $x = a_i$。
设 $a_j = a_j \oplus x$ 为所有 $1 \leq j \leq |a|$，其中 $\oplus$ 表示按位 XOR 运算。
从数组中删除 $a_i$。

可以看出，经过 $n-1$ 次操作后，数组中只剩下一个元素。你的任务是确定如果以最佳方式进行运算，最后剩下的这个元素的最大可能值。

思路

我们有一个数组 $a = [a_1, a_2, a_3, \ldots, a_n]$。假设我们从左到右进行操作，看看操作对数组有什么影响。

第一次操作后，数组变成了 $a = [a_2 \oplus a_1, a_3 \oplus a_1, \ldots, a_n \oplus a_1]$。注意，第一个元素被删除了。

第二次操作后，数组变为 $a = [(a_3 \oplus a_1) \oplus (a_2 \oplus a_1), \ldots, (a_n \oplus a_1) \oplus (a_2 \oplus a_1)]$。根据提示 $2$，这与 $a = [a_3 \oplus a_2, \ldots, a_n \oplus a_2]$ 相同。

请注意，这个新数组与 $a_1$ 无关。事实上，经过任意两次操作后，第一次操作中选择的元素值并不重要。再进一步：在进行三次运算时，前两次运算中选择的元素也无关紧要。归纳起来，唯一重要的是我们在最后一次操作中选择的元素（假设为 $x$），以及我们从未在一次操作中选择的元素（假设为 $y$）。数组的最终值将是 $x \oplus y$。

由于 $n$ 的值很小，因此只需强行求出所有可能的数对 $(x, y)$ 即可。答案就是所有数对中的最大值 $a_x \oplus a_y$。

F. The 67th Tree Problem

题意

给你两个整数 $x$ 和 $y$。这个问题有多重测试。

您的任务是构建一棵有 $x + y$ 个节点，根节点为 $1$ 的树，这样：

  • 树中恰好有 $x$ 个节点的子树 $^{\text{∗}}$ 大小为偶数。
  • 树中正好有 $y$ 个节点的子树大小为奇数。

如果存在多个有效的树，可以输出其中任何一个。如果没有这样的树，则输出 NO。

$^{\text{∗}}$ 顶点 $u$ 的子树是在通往根的简单路径上经过 $u$ 的所有顶点的集合（包括 $u$ 本身）。

思路

目标：让 $x$ 个节点深度为奇数。

方法：

  1. 先搭一条”梯子”（链）：1-2-3-4-5-…
    • 深度：0 1 2 3 4 …
    • 奇偶：偶奇偶奇偶 …
  2. 梯子上的奇数深度节点正好每隔1个出现，要 $x$ 个？那就让梯子长度 $\approx 2x$。
  3. 剩下的节点全挂在梯子顶端，它们深度相同，奇偶性一样，好控制！

G. The 67th Iteration of “Counting is Fun”

题意

给你两个整数 $n$ 和 $m$，以及一个长度为 $n$ 的数组 $b$（$0 \le b_i < m$）。保证从 $0$ 到 $m-1$ 的每个整数在 $b$ 中至少出现一次。

一群 $n$ 人站成一排，站在编号为 $1, 2, \dots, n$ 的固定位置上。他们都想坐下，但有些人太不合群，无法立即坐下。

对于长度为 $n$（$0 \le a_i < n$）的任意数组 $a$，每个人 $i$ 的社交尴尬程度为 $a_i$。考虑在离散时间单位 $0, 1, 2, \dots$ 中的座位变化过程如下：

如果 $a_i = 0$，则 $i$ 将在 $0$ 时间坐下。
否则（$a_i > 0$），如果满足以下两个条件，则 $i$ 将在时间单位 $t$ 开始时坐下：

  • 至少有 $a_i$ 人已经在时间 $t$ 之前严格坐下。
  • 至少有一个人的邻居（$i-1$ 或 $i+1$，如果存在的话）已经在时间 $t$ 之前严格坐下。

在每个时间单位开始时，每个站立的人都会检查这些条件。所有满足条件的人同时坐下，然后进入下一个时间单位。

对于每个 $i$，值 $b_i$ 表示 $i$ 坐下的时间单位。你的任务是确定有多少个不同的数组 $a$ 与给定的数组 $b$ 一致。由于这个数字可能非常大，因此输出它的模数 $676767677$。

思路

如果有 $b_i > 0$ 的人 $i$ 没有邻座有更小的 $b_i$，那么他们永远不会坐下，因此没有有效的 $a$，所以我们输出 $0$。

我们可以发现，只要与数组 $b$ 一致，所有 $a_i$ 都是相互独立的。这意味着我们可以分别计算出每个 $i$ 可能的 $a_i$ 个数。

假设 $c_t$ 是在 $t$ 或更早时间坐下的人数。这就是 $b_i \le t$ 所在位置的所有人数。对于时间 $t$，我们可以用前缀和来计算 $c_t$，因为它是 $c_{t-1}$ 和在时间 $t$ 坐下的人数之和。

要让一个人在 $t$ 时间坐下，必须满足两个条件：

  1. 至少有 $a_i$ 人在时间 $t$ 之前已经严格坐下。
  2. 至少有一位邻居（$i-1$ 或 $i+1$，如果他们存在的话）已经在时间 $t$ 之前严格坐下。

现在我们可以根据哪个条件最先出现，把人分成 $2$ 种类型。

类型 A：先满足条件 $2$。这意味着他们在等待某个邻居坐下，这样他们就可以坐下了。如果邻居最早坐下的时间恰好比自己坐下的时间少 $1$，那么这个人就属于这种类型。这意味着他们的 $a_i$ 可以是 $1$ 和 $c_{t-1}$ 之间的任意值，因此有 $c_{t-1}$ 个可能值。

类型 B：先满足条件 $1$。这意味着他们正在等待足够多的人坐下。如果一个人坐下的时间至少比其邻居最早坐下的时间多 $2$，那么他就属于这种类型。由于他们没有在 $t-1$ 时间坐下，他们的 $a_i$ 必须大于 $c_{t-2}$。这意味着它必须介于 $c_{t-2}+1$ 和 $c_{t-1}$ 之间，即有 $c_{t-1} - c_{t-2}$ 种可能性。

我们可以遍历所有的人 $i$，查看他们的邻居中最早坐下来的人是哪种类型，然后计算他们有多少个有效的 $a_i$。

最后，我们可以将每个人的有效 $a_i$ 数相乘，然后输出模数 $676767677$。

P2168 [NOI2015] 荷马史诗

题目描述

Allison 最近迷上了文学。她喜欢在一个慵懒的午后，细细地品上一杯卡布奇诺，静静地阅读她爱不释手的《荷马史诗》。但是由《奥德赛》和《伊利亚特》 组成的鸿篇巨制《荷马史诗》实在是太长了，Allison 想通过一种编码方式使得它变得短一些。

一部《荷马史诗》中有 $n$ 种不同的单词，从 $1$ 到 $n$ 进行编号。其中第 $i$ 种单词出现的总次数为 $w_i$。Allison 想要用 $k$ 进制串 $s_i$ 来替换第 $i$ 种单词，使得其满足如下要求：

对于任意的 $1\leq i, j\leq n$ ，$i\ne j$ ，都有：$s_i$ 不是 $s_j$ 的前缀。

现在 Allison 想要知道，如何选择 $s_i$，才能使替换以后得到的新的《荷马史诗》长度最小。在确保总长度最小的情况下，Allison 还想知道最长的 $s_i$ 的最短长度是多少？

一个字符串被称为 $k$ 进制字符串，当且仅当它的每个字符是 $0$ 到 $k-1$ 之间（包括 $0$ 和 $k-1$ ）的整数。

字符串 $str1$ 被称为字符串 $str2$ 的前缀，当且仅当：存在 $1 \leq t\leq m$ ，使得 $str1 = str2[1..t]$。其中，$m$ 是字符串 $str2$ 的长度，$str2[1..t]$ 表示 $str2$ 的前 $t$ 个字符组成的字符串。

输入格式

接下来 $n$ 行，第 $i + 1$ 行包含 $1$ 个非负整数 $w_i$，表示第 $i$ 种单词的出现次数。

输出格式

输出包括 $2$ 行。

第 $1$ 行输出 $1$ 个整数，为《荷马史诗》经过重新编码以后的最短长度。

第 $2$ 行输出 $1$ 个整数，为保证最短总长度的情况下，最长字符串 $s_i$ 的最短长度。

说明/提示

【样例解释】

样例 1 解释

用 $X(k)$ 表示 $X$ 是以 $k$ 进制表示的字符串。

一种最优方案：令 $00(2)$ 替换第 $1$ 种单词， $01(2)$ 替换第 $2$ 种单词， $10(2)$ 替换第 $3$ 种单词，$11(2)$ 替换第 $4$ 种单词。在这种方案下，编码以后的最短长度为：

$$1 × 2 + 1 × 2 + 2 × 2 + 2 × 2 = 12$$

$$1 × 3 + 1 × 3 + 2 × 2 + 2 × 1 = 12$$

最长字符串 $s_i$ 的长度为 $3$ 。与最优方案相比，文章的长度相同，但是最长字符串的长度更长一些。

一种最优方案：令 $000(3)$ 替换第 $1$ 种单词，$001(3)$ 替换第 $2$ 种单词，$01(3)$ 替换第 $3$ 种单词， $02(3)$ 替换第 $4$ 种单词， $1(3)$ 替换第 $5$ 种单词， $2(3)$ 替换第 $6$ 种单词。

树链剖分 学习笔记 / 模板

前言

树剖的基本思想是将树按一定方式剖分成若干条重链，重链便转化为了序列，随后我们就可以使用线段树来维护这序列上的值。这样，树上的修改与查询就成为了序列上的修改与查询，以达到更优时间复杂度的目的。

前置知识

  • 线段树：用于维护序列上的区间修改与查询。
  • 最近公共祖先 (LCA) 的相关定义。
  • 链式前向星：用于存储树图。

树剖基本内容

相关定义

名称 含义
重儿子 父节点的儿子中，子树大小最大的节点（只有一个）
轻儿子 非重儿子的节点
重边 父节点与其重儿子连成的边
轻边 父节点与其轻儿子连成的边
重链 由多条重边连成的链（叶子节点单独成链）

性质：

  1. 当前节点 x 每次向下走一条轻边到达轻儿子 y，子树大小至少除以 2。
  2. 每条重链的链顶一定是轻儿子。
  3. 任意两点的路径可以被不超过 O(log n) 条链覆盖。

详细实现

我们使用两次 DFS 实现树剖，需要维护以下数组：

数组 含义
siz[i] 子树大小
dep[i] 节点深度
fa[i] 父节点
hson[i] 重儿子
top[i] 当前节点所在重链的链顶
id[i] 节点在序列上的新编号（DFS序）
id_val[i] 新序列上第 i 个数的值

第一遍 DFS：维护 siz, dep, fa, hson

第二遍 DFS：维护 top, id, id_val

注意：重儿子优先遍历，使得同一条重链上的节点在序列中连续。

题意操作（线段树维护）

约定：

  • change(l, r, z)：区间加
  • query(l, r)：区间求和

操作1：路径上所有节点加 z

操作2：查询路径上节点和

操作3：子树内所有节点加 z

操作4：查询子树内节点和

时间复杂度

操作 时间复杂度
路径修改/查询 O(log² n)
子树修改/查询 O(log n)
总复杂度 O(m log² n)
