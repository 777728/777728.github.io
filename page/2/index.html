2026-04-18

C：

基本设定

  • 有 N 种物品（编号 1 到 N）
  • 高桥最初只拥有物品 1
  • 有 M 个朋友，第 i 个朋友可以这样交换：
    • 给他物品 Aᵢ → 他给高桥物品 Bᵢ

问题

求高桥最终能获得多少种不同的物品（包括初始的物品 1）

关键理解

这是一个有向图可达性问题：

  • 每个朋友代表一条有向边 A → B
  • 初始节点：1
  • 问从节点 1 出发，能到达多少个不同的节点

错误

  • 用并查集来处理这个问题是错误的，因为并查集适用于无向图的连通性，而这里是有向图的可达性问题。

题意

$\hspace{15pt}$请你构造 $n$ 个不重复的点（坐标均为整数），使得这些点中恰好有 $k$ 对相邻的点。

思路

📌 题目目标

构造一个二维平面上的点集，满足：

  • 使用点数 ≤ n
  • 恰好有 k 对相邻点（曼哈顿距离 = 1）

🔑 核心思路

1️⃣ 贪心 + 二分：先构造最大满网格

  • 一个 l × l 的实心正方形网格，内部相邻边数为：
  • 二分查找最大的 l，使得 2*l*(l-1) ≤ k，优先用紧凑结构”批量”消耗 k

2️⃣ 剩余 k 的处理：边缘延伸

满网格构造后，若还有剩余 k，通过在边缘挂点来精确消耗：

剩余 k 特征 构造策略 每次消耗
k 为奇数 且 k > 2l-1 先孤立挂1点 (0,1) k-=1, n-=1
k 为偶数 从 (l+1,l+1) 向左/上延伸，新点同时邻2个已有点 k-=2, n-=1
k 为奇数（小值） 从 (1,l+1) 向上单链延伸 k-=2, n-=1（首点 -1）

✅ 关键技巧：让新点同时连接2个已有点，可高效消耗 k（1点换2边）

3️⃣ 孤立点兜底

若 n 还有剩余，输出坐标稀疏的孤立点（如 (1e9, 偶数)），确保：

  • 不产生额外相邻边
  • 消耗完所有可用点数

AtCoder Beginner Contest 452

2026-04-15

D

题意

给定字符串 S 和 T，问 S 的所有非空子串中，有多少个子串不以 T 作为子序列（不要求连续）。

关键点：

  • 子串是连续的，子序列可以不连续。
  • 两个子串只要位置不同就算不同（即使内容相同）。
  • 长度限制：|S| ≤ 2×10^5，|T| ≤ 50。

换句话说，要统计所有子串 [l, r]（1 ≤ l ≤ r ≤ |S|），使得 不存在一种方式从 S[l..r] 中按顺序挑出字符等于 T。

更直白地：子串内不能按顺序匹配出整个 T。

思路

答案 = 总子串数 − 包含 T 的子串数

关键步骤

  1. 预处理 nxt[i][c]：从位置 i 开始，字符 c 首次出现的位置（O(26n)）

  2. 对每个左端点 i：

    • 贪心匹配 T：pos = nxt[pos][t[j]]; pos++
    • 若匹配成功，cnt += n - pos + 1（以 i 为左端点的合法右端点个数）

  3. 输出：n*(n+1)/2 - cnt

代码

E

题意总结

给定两个正整数序列 $A=(A_1,\dots,A_N)$ 和 $B=(B_1,\dots,B_M)$，求：

$$\sum_{i=1}^{N} \sum_{j=1}^{M} A_i \cdot B_j \cdot (i \bmod j) \pmod{998244353}$$

约束：$N,M \le 5\times 10^5$，$A_i,B_j \le 5\times 10^5$

解题思路

核心观察

  1. 模运算展开：$i \bmod j = i - j \cdot \lfloor i/j \rfloor$

  2. 按商分组：对于固定的 $j$，当 $\lfloor i/j \rfloor = k$ 时，$i$ 的范围是 $[jk, \min(j(k+1)-1, N)]$，在此范围内 $i \bmod j = i - jk$

  3. 区间求和转化：
    $$\sum_{i=l}^{r-1} A_i \cdot (i - jk) = \sum_{i=l}^{r-1} A_i \cdot i - jk \cdot \sum_{i=l}^{r-1} A_i$$

  4. 前缀和优化：预处理两个前缀和数组，可在 $O(1)$ 时间内计算区间和：

    • sum[x] = $\sum_{k=0}^{x-1} A_k$
    • sum2[x] = $\sum_{k=0}^{x-1} A_k \cdot k$

  5. 复杂度分析：对于每个 $j$，$k$ 的范围是 $0 \sim \lfloor N/j \rfloor$，总枚举次数为 $\sum_{j=1}^M \lfloor N/j \rfloor = O(N \log N)$（调和级数）

代码注释解析

时间 $O(N \log N + M)$
空间 $O(N + M)$

AtCoder ABC 452 F - Interval Inversion Count

📋 题意简述

给定长度为 $N$ ($N \leq 5 \times 10^5$) 的排列 $P$ 和整数 $K$，求有多少个区间 $[l, r]$ 满足：该区间内子序列的逆序对数量恰好等于 $K$。

💡 解题思路

核心转化

直接统计「恰好 $K$ 个逆序对」较难，转化为：
$$\text{ans} = f(K) - f(K-1)$$
其中 $f(k)$ 表示逆序对数 $\leq k$ 的区间数量。

单调性 & 双指针

关键观察：区间缩短，逆序对数不会增加。

  • 若 $[l, r]$ 逆序对 $\leq K$，则 $[l+1, r]$ 和 $[l, r-1]$ 也 $\leq K$
  • 因此对每个左端点 $l$，满足条件的右端点 $r$ 具有单调性 → 可用滑动窗口（双指针）

逆序对动态维护（Fenwick Tree）

窗口 $[l, r)$ 内的逆序对动态更新：

操作 逆序对变化 BIT 查询
右扩 $r$ $+$ 窗口中 $> P[r]$ 的元素个数 bit.sum(P[r], N)
左缩 $l$ $-$ 窗口中 $< P[l]$ 的元素个数 bit.sum(0, P[l])

💻 代码详解（带注释）

🔑 关键细节

  1. BIT 查询范围：

    • bit.sum(a, b) 查询值域 $[a, b)$ 的元素个数
    • 加 $P[r]$ 时：查 $[P[r], N)$ 即大于 $P[r]$ 的值 → 新增逆序对
    • 减 $P[l]$ 时：查 $[0, P[l])$ 即小于 $P[l]$ 的值 → 减少逆序对

  2. 时间复杂度：$O(N \log N)$

    • 双指针每个元素最多进出窗口一次：$O(N)$
    • 每次 BIT 操作 $O(\log N)$

  3. 边界处理：$K=0$ 时 $f(-1)=0$，需特判避免下溢

🧪 样例验证（Sample 1）

题意总结

定义：一个正整数序列 $X$ 被称为 221 序列，当且仅当对其游程编码（Run-Length Encoding, RLE）后的每一段 $(值, 长度)$ 都满足 值 = 长度。

  • 例如 (2,2,3,3,3,1) 是 221 序列，因为 RLE 为 (2,2), (3,3), (1,1)，每段值都等于长度。
  • (1,1) 不是，RLE 为 (1,2)，$1 \ne 2$。

问题：给定长度为 $N$ 的序列 $A$（$A_i \in [1,9]$），求 $A$ 中有多少个 不同的非空连续子序列 是 221 序列。

  • 即使子序列出现在不同位置，只要序列内容相同只计一次。

约束：$N \le 5 \times 10^5$，$A_i \le 9$。

G - 221 Substring

解题思路

1. 游程编码分析

对 $A$ 进行游程编码，得到块序列 $(v_1, m_1), (v_2, m_2), \dots, (v_k, m_k)$，其中 $v_i$ 是值，$m_i$ 是连续出现次数。

一个子序列是 221 序列，当且仅当它对应 RLE 的一段区间 $[l, r]$，且满足：

  • 内部块（$l < i < r$）：必须完整保留，且 $v_i = m_i$。
  • 边界块（$i = l$ 或 $i = r$）：可以截取一部分。若截取长度为 $L$，需满足 $L = v_i$，这要求 $v_i \le m_i$。
    • 左边界可取后缀长度为 $v_l$ 的部分。
    • 右边界可取前缀长度为 $v_r$ 的部分。

因此，一个合法的 221 子序列由值序列 $(v_l, v_{l+1}, \dots, v_r)$ 唯一确定，且需满足：

  • 对所有 $i \in (l, r)$，$v_i = m_i$。
  • $v_l \le m_l$ 且 $v_r \le m_r$。
  • 若 $v_i > m_i$，该块无法参与任何 221 序列。

2. 问题转化

构造一个新序列 $T$，将每个 RLE 块映射如下：

  • 若 $v_i > m_i$：映射为 (0)（表示断裂，无法使用）。
  • 若 $v_i = m_i$：映射为 (v_i)（可作为内部或边界，可连接）。
  • 若 $v_i < m_i$：映射为 (v_i, 0, v_i)。
    • 第一个 $v_i$ 可作为右边界（与其左侧连接）。
    • 第二个 $v_i$ 可作为左边界（与其右侧连接）。
    • 中间的 0 阻止左右穿透，保证该块不能同时作为左右边界或作为内部块。

关键结论：$A$ 中不同的 221 子序列与 $T$ 中 不包含 0 的不同子数组 一一对应。

  • 因为 221 序列的结构由值序列唯一确定，计数不同 221 序列等价于计数 $T$ 中不含 0 的不同子数组。

3. 算法实现

问题转化为：求序列 $T$ 中不含 0 的不同子数组个数。

使用 后缀数组 (Suffix Array, SA) 和 LCP 数组：

  • 设 zerofree[i] 表示从位置 $i$ 开始向后连续不含 0 的最大长度。
  • 对后缀数组中第 $k$ 个后缀（起始位置为 SA[k]），其能贡献的不含 0 的新子串数量为：
    $$\max(0, \text{zerofree}[\text{SA}[k]] - \text{LCP}[k])$$
    其中 LCP[k] 是该后缀与前一个后缀的最长公共前缀长度，需减去重复部分。
  • 答案为所有后缀贡献之和。

复杂度：

  • 构造 $T$：$O(N)$。
  • 后缀数组与 LCP：$O(|T|)$（利用字母表大小 $\sigma=10$ 可线性构造）。
  • 总时间复杂度 $O(N)$，空间 $O(N)$。

代码注释解析

代码关键点说明

  1. maximals 构造逻辑：
    • 当遇到 val < len 的块时，区间包含它并停止扩展，但 idxlen 更新为 jlen（未越过该块）。
    • 下一轮循环会再次从该块开始，使其作为新区间的起点。
    • 效果：val < len 的块会在 s 中出现两次，分别作为前一个块的结尾和后一个块的开头，中间被 0 隔开，完美实现 (v, 0, v) 转换。
  2. count_zerofrees：
    • 利用后缀数组去重，r - l 即为当前后缀贡献的新子串数。
    • max_zerofree 限制了子串不能包含 0。
  3. 复杂度：整体 $O(N)$，可轻松通过 $N=5\times 10^5$。

代码

e

题意

蓝桥杯2026

2026-04-14

B

题意

这道题要求计算在 $2048 \times 2048$ 的网格中，有多少种将每个格子填为 $L$ 或 $Q$ 的方案，满足三个条件：

  1. 每行 $Q$ 的个数为奇数
  2. 每列 $Q$ 的个数为奇数
  3. 任意 $2\times2$ 子区域中 $Q$ 的个数为奇数

你需要输出结果对 $998244353$ 取模。

思路

打表

n 为奇数时只有 1 种方案，n 是 4 的倍数时，注意到 32 和 8192 都是 2 的幂，可以归纳出来这种时候是 2 ^ (2n−3)，其他情况下的 n 的答案是 0。题目要求的 n=2048，是 4 的倍数，所以答案就是 2 ^ 4093 取模。

推理

由于每个 2×2 的矩阵的和都是奇数，即 a 1,1​+a 1,2​+a 2,1​+a 2,2​是奇数且 a 2,1​+a 2,2​+a 3,1​+a 3,2​也是奇数，于是我们知道 a 1,1​+a 1,2​和 a 3,1​+a 3,2​同奇偶。

推广出来，我们发现 a i,j​+a i+1,j​和 a i,j−2​+a i+1,j−2​同奇偶，且 a i,j​+a i,j+1​与 a i+2,j​+a i+2,j+1​同奇偶。

如果我们已知第一行和第一列的所有数，根据这个规律可以把整个矩阵填满（首先 a 1,1​,a 1,2​,a 2,1​已知，奇偶性可以算出 a 2,2​，然后利用隔行隔列同奇偶的性质可以填满整个矩阵，可以手玩一下）。

同时如果保证第一行的和是奇数的话，则由于第三行每相邻两个数的和的奇偶性和第一行相同，因此所有奇数行的和肯定也是奇数；由于每个 2×2 的矩阵的和都是奇数，因此前两行的和一定是偶数（前两行可以分割成 1024 个 2×2 的矩阵），因此第二行的和也是奇数，因此偶数行的和也是奇数。

同理只要第一列的和是奇数，所有列的和都是奇数。

怎么保证第一行第一列的和是奇数？只填 2047 个，算算奇偶性决定最后一个怎么填。

于是一共有 2047+2047−1=4093 个位置可以随便填，答案就是 2 4093 。减一因为 a 1,1​被算了两次。

题意

题目要求：

有 $N$ 根木材，长度分别为 $A_i$，最多可以切 $K$ 刀（每次切一根成两段，长度正整数）。

目标：找到最小的 挡板间距 $L$，使得存在一种切割方案，切割后 所有木段的长度都不超过 $L$。

  • $L$ 越小，要求切割后每段越短，需要的切割次数可能越多。
  • 要保证切割次数不超过 $K$。

核心思路

  • 给定 $L$，可以计算每根木材至少需要切几刀才能让每段长度 $\le L$：
    长度 $A_i$ 的木材，切 $t_i = \lceil A_i / L \rceil - 1$ 刀，得到 $\lceil A_i / L \rceil$ 段。
  • 总刀数 $T = \sum t_i$ 必须 $\le K$。
  • 要找最小的 $L$，使 $T \le K$。

输入：

  • $N, K$
  • $A_1 \dots A_N$

输出：

  • 最小 $L$

数据范围：
$N$ 最大 $10^6$，$K$ 最大 $10^9$，$A_i$ 最大 $10^9$。

E

题目理解

我们有：

  • 硬盘扇区：编号 ( 1, 2, \dots, N )
  • 第 (i) 个扇区的数据量：( a_i = C \times i )
  • 我们可以选择一个连续的扇区区间 ([l, r]) 读取，会得到数据总量：
    [
    \sum_{i=l}^r C \times i = C \times \frac{(l+r)(r-l+1)}{2}
    ]
  • 每次读取是连续区间，不能重复读取（每个扇区最多读一次）
  • 要求：读出的数据总量 恰好等于 ( W )
  • 目标：最少指令次数（即最少区间数）

思路

首先如果 w 不是 c 的倍数肯定 −1，否则令 x= c w​。

如果 1+2+⋯+n= 2 n(n+1)​<x，也是无解。

如果 w=0，答案是 0。（赛时代码好像没考虑这个，寄）

否则，下面证明答案最多是 2。

由于
2 n(n+1)​≥x，因此一定存在一个 1≤m<n，使得：

1+2+⋯+m= 2 m(m+1)​≤x； 1+2+⋯+(m+1)>x。 我们可以证明 y=1+2+⋯+(m+1)−x=(m+1)−[x−(1+2+⋯m)]≤(m+1)，我们把 (m+1) 补上，再把 y 删去，最多两个区间。

所以答案不是 1 就是 2。开 map 记录一下前缀和，即可判断一个区间是否可行，可行则为 1，否则为 2。

F

题意

午高峰有 $N$ 个订单，必须按固定顺序依次配送，可分成若干批次。站点有 $M$ 种交通工具，每种具属性 $A_i$（路途耗时系数）和 $B_i$（拥挤系数）。

对于一批含 $L$ 个订单，选用第 $i$ 种工具，耗时为：

  1. 路途：$L \times A_i$
  2. 取餐：$\frac{L(L-1)}{2} \times B_i$（处理挤压关系）
  3. 折返：首批为 $0$，之后每新开一批需 $+X$

求送完 $N$ 单的最小总耗时。

输入：$N, M, X$ 及 $M$ 行 $(A_i, B_i)$
输出：最小总耗时（整数）
范围：$N, M \le 5000$，$A_i, B_i, X \le 10^9$

思路

1. 预处理最优单车耗时

对每个可能的批次长度 $l \in [1, N]$，枚举 $M$ 种车，计算该长度下的最小基础耗时（不含折返）：
$$c[l] = \min_{1 \le i \le M} \left( l \cdot A_i + \frac{l(l-1)}{2} \cdot B_i \right)$$

2. 动态规划

设 $dp[i]$ 表示送完前 $i$ 单的最短时间。

枚举最后一批的起点 $j$（即最后一批为 $[j+1, i]$，长度 $i-j$）：

  • 若 $j=0$（整批为第一批）：耗时 $c[i]$
  • 若 $j>0$：耗时 $dp[j] + c[i-j] + X$

转移方程：
$$dp[i] = \min\left(c[i],\ \min_{1 \le j < i} (dp[j] + c[i-j] + X)\right)$$

技巧：初始化 dp[i] = c[i] 可自然处理 $j=0$ 的边界，无需特判。

3. 复杂度

  • 预处理：$O(NM)$
  • DP：$O(N^2)$
  • 总计：$O(N^2 + NM)$，$N,M \le 5000$ 可过

G

题意

给定长度为 $N$ 的序列 $A$，选择三个分割点 $1 \le i < j < k < N$，将序列切成 4 段非空连续子段。

目标：最大化
$$|\text{sum}_1| + |\text{sum}_2| + |\text{sum}_3| + |\text{sum}_4|$$

其中 $\text{sum}_t$ 表示第 $t$ 段的元素和。

输入：$N$ 及序列 $A_1 \dots A_N$
输出：最大绝对值和
范围：$4 \le N \le 10^5$，$|A_i| \le 10^9$

思路

方法一：枚举符号 + 线性 DP（O(16N)，推荐⭐）

核心洞察

绝对值 $|x| = \max(x, -x)$。若我们提前假设每段和的符号（正/负），绝对值即可直接去掉。

枚举 4 段符号的所有 $2^4=16$ 种组合，对每种组合：

  • 若第 $t$ 段假设符号为 $sgn[t] \in {+1,-1}$
  • 目标变为：$sgn[1]\cdot S_1 + sgn[2]\cdot S_2 + sgn[3]\cdot S_3 + sgn[4]\cdot S_4$

为什么正确？

若某段实际和与假设符号相反，该项贡献会比真实 $|x|$ 小 $2|x|$，结果一定 $\le$ 真实答案。枚举所有组合取 $\max$，必能命中最优符号分配。

DP 转移

dp[i][k]：前 $i$ 个数分成 $k$ 段，在当前符号组合下的最大带符号和。

$$dp[i][k] = \max(dp[i-1][k],\ dp[i-1][k-1]) + sgn[k] \cdot A[i]$$

  • dp[i-1][k]：$A[i]$ 接在第 $k$ 段末尾
  • dp[i-1][k-1]：$A[i]$ 作为第 $k$ 段开头

复杂度

$16 \times 4 \times N = O(64N)$，常数极小，代码简洁。

方法二：拆绝对值 + 前缀最大值（O(4N)，理论最优）

核心转化

设前缀和 $S[i] = \sum_{j=1}^i A[j]$，第 $[j+1, i]$ 段和为 $S[i]-S[j]$。

转移方程：
$$f[i][k] = \max_{j < i} \left( f[j][k-1] + |S[i] - S[j]| \right)$$

拆绝对值：

  • 若 $S[i] \ge S[j]$：$f[j][k-1] - S[j] + S[i]$
  • 若 $S[i] < S[j]$：$f[j][k-1] + S[j] - S[i]$

只需维护两个全局最大值：

  • max1 = max(f[j][k-1] - S[j])
  • max2 = max(f[j][k-1] + S[j])

复杂度

$4 \times N = O(N)$，理论最快，但边界处理需谨慎。

方法三：线段树优化（O(N log N)，通用扩展）

适用场景

当题目增加区间修改、动态查询等复杂条件时，前缀最大值法失效，线段树可灵活应对。

核心思想

仍基于拆绝对值，用线段树按 $S[j]$ 的值域维护：

  • m1: $\max(f[j][k-1] - S[j])$（用于 $S[i] \ge S[j]$）
  • m2: $\max(f[j][k-1] + S[j])$（用于 $S[i] < S[j]$）

配合离散化 + 懒标记实现高效查询更新。

H

题意

给定长度为 $n$ 的基因序列生成规则和长度为 $m$ 的易感模式串 $s$。

  1. 概率分布：序列第 $i$ 位字符为 A/T/G/C 的概率已知（模 $998244353$ 意义下），且各位独立。
  2. 目标：计算随机生成的长度为 $n$ 的序列中，包含子串 $s$ 的概率。
  3. 输出：将该概率值在模 $998244353$ 意义下的结果输出（即若概率为 $p/q$，输出 $x$ 使得 $x \cdot q \equiv p \pmod{998244353}$）。
    数据范围：$1 \le m \le n \le 3000$。

思路

思路一：KMP 自动机 + 补集法（最主流、推荐）

核心思想：
正难则反。直接计算“包含子串”的概率较难处理重叠情况，因此计算不包含易感序列的概率 $P_{no}$，最终答案为 $1 - P_{no}$。

  1. 构建 KMP 自动机：预处理 trans[state][char]，表示当前匹配了 state 个字符，下一个字符是 char 时，新的匹配长度是多少。
  2. DP 状态：$dp[i][j]$ 表示生成了前 $i$ 个字符，当前 KMP 匹配长度为 $j$ ($0 \le j < m$)，且从未完全匹配过模式串的概率。
  3. 转移：枚举下一位字符，若新状态 $k < m$，则累加；若 $k = m$，说明此时首次匹配成功，该路径不计入 $dp$（因为它变成了“易感”人群）。
  4. 答案：$1 - \sum_{j=0}^{m-1} dp[n][j]$。

代码实现：

思路二：KMP 自动机 + 直接法（标记是否已匹配）

核心思想：
不采用补集，直接在 DP 状态中记录“是否已经包含过易感序列”。

  1. DP 状态：$dp[i][j][k]$，其中 $i$ 是长度，$j$ 是当前 KMP 匹配长度，$k \in {0, 1}$ 表示是否已经出现过完整子串（0:否，1:是）。
  2. 转移：
    • 若当前 $k=0$ 且转移后状态达到 $m$，则新状态 $k’=1$。
    • 若当前 $k=1$，无论转移到哪个状态，新状态 $k’$ 仍为 $1$。
    • 若当前 $k=0$ 且转移后状态 $<m$，则新状态 $k’=0$。
  3. 答案：$\sum_{j} dp[n][j][1]$。

代码实现：

思路三：AC 自动机法（通用性强，适合多模式串）

核心思想：
虽然本题只有一个模式串，但 AC 自动机是处理此类问题的通用模板。

  1. 建 Trie 树：将易感序列插入字典树。
  2. 构建 Fail 指针：类似 KMP，但在 Trie 图上构建。
  3. 标记危险节点：如果一个节点代表完整串，或者其 Fail 指向的节点代表完整串，则该节点标记为“已找到”。
  4. DP：$dp[i][node][flag]$，在 AC 自动机的节点 node 上，是否已触发 flag。
  5. 特点：代码量稍大，但对于理解自动机原理很有帮助。

代码实现：

思路四：暴力预处理 Border + DP（无需显式 KMP 自动机）

核心思想：
对于数据范围较小 ($n,m \le 3000$) 的情况，可以不写复杂的自动机构建逻辑。

  1. 预处理 f[i][c]：表示当前匹配了 $i$ 个字符，下一个字符是 $c$ 时，能匹配到的最长前缀长度。这可以通过暴力枚举 border 来实现。
  2. DP：$dp[i][j]$ 表示前 $i$ 位，后缀匹配了 $j$ 位（且 $j<m$）的概率。
  3. 转移：直接查表 f[j][c] 进行转移。如果结果为 $m$，则计入答案（或根据补集法丢弃）。

  • 首选：思路一（KMP 自动机 + 补集法）。逻辑最清晰，代码简洁，不易出错，时间复杂度最优。
  • 备选：思路二（直接法）。如果你更习惯正向思考“是否包含”，可以使用此方法，状态多一维但逻辑直观。
  • 学习：思路三（AC 自动机）。如果你想掌握处理多模式串匹配的通用技能，值得学习。
